# Lost Book Robot - OCR处理系统文档

## 项目概述

Lost Book Robot是一个用于自动检测图书馆书籍错位的机器人系统。该系统通过OCR技术识别书脊上的Call Number，验证书籍是否按照LC（Library of Congress）分类规则正确排列。

## 系统架构

### 数据流程
```
原始图像 → OCR引擎 → 原始文本块 → 空间聚类 → Call Number重建 → 数据融合 → 排序验证 → 结果输出
```

### 核心组件

#### 1. OCR处理器 (OCRProcessor)
- **功能**：系统的主控制器，协调整个处理流程
- **输入**：OCR引擎返回的原始结果
- **输出**：结构化的Call Number列表和错位检测结果

#### 2. 空间聚类器 (SpatialClusterer)
- **功能**：将OCR返回的分散文本块按空间位置分组
- **算法**：基于中心点距离的聚类算法
- **参数**：距离阈值（默认50px）
- **关键逻辑**：
  - 计算每个文本块的中心点
  - 将距离小于阈值的文本块归为一组
  - 每组对应一本书的Call Number组成部分

#### 3. LC Call Number解析器 (CallNumberParser)
- **功能**：识别和解析LC标准的Call Number格式
- **支持格式**：
  - 基础格式：`BR 330`
  - 完整格式：`BR 330 E5 1955 v.28`
- **验证逻辑**：
  - 字母部分必须在有效的LC分类范围内
  - 数字部分必须符合LC编号规则
  - Cutter号、年份、卷号的可选验证

#### 4. 数据融合器 (DataFusion)
- **功能**：合并多次扫描的结果，提高准确性
- **策略**：
  - 位置指纹：基于坐标网格的位置编码
  - 置信度加权：优先选择高置信度的识别结果
  - 投票机制：多次扫描结果的一致性验证
- **时序处理**：维护扫描历史，支持增量更新

#### 5. 排序验证器 (SortingValidator)
- **功能**：检查书籍排列顺序是否符合LC规则
- **比较规则**：
  1. 字母部分（A > B > C...）
  2. 数字部分（100 > 99 > 98...）
  3. Cutter号（E5 > E4 > E3...）
  4. 年份（1955 > 1954...）
  5. 卷号（v.29 > v.28...）
- **错误检测**：
  - 相邻书籍顺序错误
  - 序列中的大间隙
  - 重复的Call Number

## 核心挑战与解决方案

### 挑战1：Call Number碎片化
**问题**：OCR引擎返回的是分散的文本块
**示例**：`["BR", "330", "E5", "1955", "v.28"]`
**解决方案**：
- 空间位置聚类：将相近的文本块归为一组
- 模式匹配：使用正则表达式识别LC格式
- 多重组合尝试：不同的文本块组合方式

### 挑战2：多次扫描的数据一致性
**问题**：机器人移动过程中，同一本书可能被多次扫描
**解决方案**：
- 位置指纹：基于坐标网格的统一位置编码
- 时序融合：维护扫描历史，合并多帧结果
- 置信度提升：通过多次扫描验证提高准确性

### 挑战3：LC分类规则的复杂性
**问题**：LC Call Number有特定的排序规则
**解决方案**：
- 专门的解析器：严格按照LC标准解析
- 多级比较：按照字母→数字→Cutter→年份→卷号的顺序
- 容错机制：处理格式不一致的情况

## 实现路径

1. **输入处理**：解析openOCR的输出格式
2. **空间聚类**：实现文本块分组算法
3. **Call Number重建**：从文本块组合出完整的Call Number
4. **基础验证**：单次扫描的准确性测试

### Phase 2：多次扫描融合
1. **时序数据管理**：维护扫描历史记录
2. **位置指纹生成**：实现统一的位置编码
3. **数据融合算法**：实现投票和加权机制
4. **性能优化**：处理大规模扫描数据

### Phase 3：排序验证
1. **LC比较器实现**：完整的Call Number比较逻辑
2. **错误检测算法**：识别错位和缺失的书籍
3. **灵敏度调优**：平衡误报率和漏报率
4. **结果报告**：生成详细的错位报告

### Phase 4：系统集成
1. **机器人接口**：与移动平台的控制系统集成
2. **实时反馈**：提供即时的错位检测反馈
3. **错误处理**：处理各种异常情况
4. **性能优化**：提高处理速度和准确性

### 性能要求
- **处理速度**：单帧处理时间 < 1秒
- **准确性**：Call Number识别准确率 > 95%
- **容错性**：支持部分遮挡和光照变化
- **扩展性**：支持多机器人并行工作

## 配置参数

### 空间聚类参数
- `distance_threshold`: 50.0 (文本块距离阈值)
- `confidence_threshold`: 0.7 (置信度阈值)

### 数据融合参数
- `confidence_weight`: 0.8 (置信度权重)
- `historical_decay`: 0.9 (历史数据衰减因子)
- `vote_threshold`: 0.6 (投票阈值)

### 排序验证参数
- `max_gap_size`: 10 (最大允许间隙)
- `min_confidence`: 0.8 (最小置信度要求)
- `tolerance_level`: 2 (容错级别)
